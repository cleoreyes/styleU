App Description

Our Users

Our main users will be those that listen and enjoy usic of the K-pop genre. We are looking to minimize their time looking up songs to which they may not have all the information of, generate playlist based off of their preferences, and reduce the need for customizing their current playlist.

  1. Below are some the possible interactive features we intend to implement
    • Allow users to input a K-pop song or artist, and based off those inputs, it generates similar music.
    • Login feature using Google oAuth or account creation with email and password to allow users to save their music session history.
    • Implement EmailJS to allow users to transfer their outputs to a designated email.
    • Natural language comprehension using OpenAI’s whisper and process that through ChatGPT to create functional commands to navigate through the frontend UI or either find songs based off matched lyrics.
  2. The information that we will be using is K-pop data taken from a dataset in kaggle and from an existing database found at dbkpop. Some data values include the artist/group, album, song, date released, and hopefully other features like lyrics. We intend to import this data from a csv which can be found from dbkpop which was mentioned earlier and kaggle-kpop

Users can specify their favorite artists, albums, or even individual group members to receive recommendations tailored to their tastes.
